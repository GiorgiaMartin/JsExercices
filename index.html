Risposte teoriche prima parte React / M5W1 1. Cos'è ReactJS? ReactJS è uno dei migliori framework per front-end. E' stato creato da Meta è utile per creare pannelli gestione "privati" non indicizzati, serve per creare un interfaccia utente. 2. ReactJS è un framework, non una libreria. Vero 3. Il file package.json contiene… Molte informazioni utili, come ad esempio l'elenco di tutte le dipendenze richieste dall'applicazione 4. create-react-app è l'unico modo possibile per creare un'applicazione React. Falso, è solo un boilerplate con strumenti già integrati, per effettuare automaticamente tutta la configurazione e le installazioni dei pacchetti necessari e avviare una nuova app React localmente, pronta per lo sviluppo. 5. Qual è il comando da lanciare nel terminale per creare una nuova create-react-app con nome “test”? mpm create-react-app test 6. Cos'è un componente React? Un blocco di logica/contenuto riutilizzabile all'interno dell'applicazione. 7. Un componente React può venire creato in tre modi: come funzione, classe o interfaccia. Falso 8. Le props sono frammenti di informazione assegnati all'invocazione di un componente React, utili al fine di rendere il componente dinamico e più riutilizzabile. Vero/Falso 9. Le props possono essere passate solamente da un componente genitore ad un componente figlio, non è possibile fare il contrario. Vero 10. Da dove possono venire recuperate le props all'interno di un componente React creato come classe? Possono essere recuperate al'interno dello oggetto 'this', dentro un sotto-oggetto chiamato 'props'.
